⚠ Storage Warning The Jetson's internal eMMC (54GB) is too small for Docker images + Ollama models (~30GB total). Move Docker's data-root and Ollama model storage to an external SSD before proceeding.

Step 0 — Prepare External Storage

Move Docker and model storage to SSD to avoid filling internal eMMC

Move Docker data-root to SSD

# Stop Docker sudo systemctl stop docker # Create directory on SSD sudo mkdir -p /media/orin/SSD4TB/docker-data # Copy existing Docker data sudo rsync -aP /var/lib/docker/ /media/orin/SSD4TB/docker-data/ # Configure Docker to use SSD sudo nano /etc/docker/daemon.json

Add or update daemon.json:

{ "runtimes": { "nvidia": { "path": "nvidia-container-runtime", "runtimeArgs": [] } }, "default-runtime": "nvidia", "data-root": "/media/orin/SSD4TB/docker-data" }

Move containerd to SSD (prevents internal storage leak)

sudo systemctl stop containerd sudo mkdir -p /media/orin/SSD4TB/containerd-data sudo rsync -aP /var/lib/containerd/ /media/orin/SSD4TB/containerd-data/ sudo rm -rf /var/lib/containerd sudo ln -s /media/orin/SSD4TB/containerd-data /var/lib/containerd # Restart services sudo systemctl start containerd sudo systemctl start docker # Verify docker info | grep "Docker Root Dir" # Should show: /media/orin/SSD4TB/docker-data

Step 1 — Build the Docker Image

A single Dockerfile installs all dependencies

Dockerfile

# MindGuard Dockerfile for NVIDIA Jetson Orin AGX FROM dustynv/l4t-pytorch:r36.2.0 ENV DEBIAN_FRONTEND=noninteractive # System dependencies RUN apt-get update && apt-get install -y --no-install-recommends \ curl zstd git wget libgl1-mesa-glx \ libglib2.0-0 libsm6 libxext6 libxrender-dev v4l-utils \ && rm -rf /var/lib/apt/lists/* # Fix blinker + downgrade numpy for PyTorch 2.2 compatibility RUN rm -rf /usr/lib/python3/dist-packages/blinker* && \ pip3 install --no-cache-dir "numpy<2" && \ pip3 install --no-cache-dir --index-url https://pypi.org/simple \ blinker "transformers<4.46" ultralytics \ opencv-python flask ollama # Install Ollama RUN curl -fsSL https://ollama.com/install.sh | sh # Download YOLO model RUN python3 -c "from ultralytics import YOLO; YOLO('yolo26n.pt')" WORKDIR /workspace EXPOSE 5000 11434 CMD ["bash"]
ℹ Key Fixes in this Dockerfile blinker — the base image ships a broken distutils version; we force-remove it before pip install.
numpy<2 — PyTorch 2.2.0 on Jetson was compiled against NumPy 1.x; NumPy 2.x causes crashes.
transformers<4.46 — Transformers v5+ requires PyTorch ≥ 2.4, but JetPack ships 2.2.0.

Build

cd /media/orin/SSD4TB/MindGuard docker build -t mindguard:jetson .

Step 2 — Run the Container

Launch with GPU runtime, camera access, and SSD volume mounts

docker run -it \ --name mindguard \ --runtime=nvidia \ --network host \ --ipc host \ --privileged \ -v /dev:/dev \ -v /media/orin/SSD4TB/MindGuard:/workspace \ -v /media/orin/SSD4TB/ollama-models:/root/.ollama \ mindguard:jetson

--runtime=nvidiaEnable GPU access inside container
--network hostOllama API + Flask web UI accessible from LAN

Pull Models

# Start Ollama server ollama serve & sleep 3 # Pull required models (~4.2GB total) ollama pull llava-phi3 # VLM for scene analysis (2.3GB) ollama pull qwen2.5:3b # Fusion agent (1.9GB)

"

Key Takeaways

The complete MindGuard pipeline — YOLO26, BERT, LLaVA-Phi3, and Qwen2.5-3B — runs entirely on a single Jetson Orin AGX with no cloud dependency. The 9.3-second full pipeline latency is sufficient for the autonomous background monitor operating at 10-second scan intervals. The text branch (keywords + BERT) remains fast at 0.14 seconds, confirming that lightweight classifiers are well-suited for edge hardware. The VLM and fusion agent account for over 98% of total latency, making them the primary optimization targets for future TensorRT acceleration.

Troubleshooting

Common issues and fixes

Blinker installation error

ERROR: Cannot uninstall 'blinker'

# Force remove before installing rm -rf /usr/lib/python3/dist-packages/blinker* pip3 install blinker

NumPy version mismatch

A module that was compiled using NumPy 1.x cannot be run in NumPy 2.x

pip3 install "numpy<2"

Transformers requires newer PyTorch

PyTorch >= 2.4 is required but found 2.2.0

pip3 install "transformers<4.46"

Ollama not using GPU

Shows inference compute: cpu and total_vram: 0 B

# Known JetPack 6 issue — Ollama's llama-server # GPU discovery crashes on Jetson unified memory. # PyTorch CUDA works fine; only Ollama is affected. # Workaround: use CPU inference (current approach) # Future fix: TensorRT export for direct GPU inference

Camera not found inside container

can't open camera by index

# Use -v /dev:/dev and --privileged when creating container docker run -it --privileged -v /dev:/dev ...

Internal storage filling up

# Check usage df -h /dev/mmcblk0p1 # Move containerd to SSD (see Step 0) sudo rm -rf /var/lib/containerd sudo ln -s /media/orin/SSD4TB/containerd-data /var/lib/containerd
